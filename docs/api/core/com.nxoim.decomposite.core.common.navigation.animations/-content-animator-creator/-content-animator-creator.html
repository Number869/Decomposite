ContentAnimatorCreator

constructor(key: String, renderUntil: Int, requireVisibilityInBackstack: Boolean, animatorScopeFactory: (initialIndex: Int, initialIndexFromTop: Int) -> T, animationModifier: (T) -> Modifier)

Parameters

key

helps optimize scope creation by preventing the creation of a new scope if one with the same key already exists.

renderUntil

controls content rendering based on its position in the stack, with 0 being the top. StackAnimator will not render an item if its position is greater than renderUntil. The animation scope implementation can be aware of renderUntil.

requireVisibilityInBackstack

manages the item's visibility in the backstack after animations are completed. If an item's position exceeds renderUntil and requireVisibilityInBackstack is true, the item will remain visible. Note that if multiple animations (e.g., fade() + scale()) are combined, and at least one has requireVisibilityInBackstack set to true, all items that do not meet renderUntil will be visible in the backstack.

animatorScopeFactory

is used to create the animation scope. Refer to contentAnimator for an example.

animationModifier

provides the animated Modifier to the content.
